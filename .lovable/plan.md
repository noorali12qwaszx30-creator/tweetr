# خطة تحسين شاشة المطبخ

## المشكلتان
1. **شريط "تم تعديل هذا الطلب"** يحتل عرض الكارت كاملاً ويأكل من مساحة عرض الأكلات (وكذلك شريط "طلب جديد").
2. عند تجاوز عدد الطلبات النشطة سعة الشبكة (18 = 6×3)، الطلبات الإضافية تختفي أسفل الشاشة لأن `<main>` فيه `overflow-auto` ولا يوجد تدوير تلقائي.

---

## التغييرات

### 1) شارة ركن بدل الشريط الكامل (`src/components/KitchenOrderCard.tsx`)
- استبدال `EditedRibbon` (الشريط الأحمر العريض) بـ **شارة عائمة صغيرة في الزاوية العلوية اليسرى** (نفس فكرة الرسم المرفوع):
  - دائرة/شارة حمراء مدمجة `absolute top-1 left-1 z-30` فيها أيقونة ✏️ + نص قصير "معدّل" مع `animate-pulse`.
  - لا تأخذ أي مساحة من قائمة الأكلات.
- نفس المعالجة لـ `FreshRibbon`: تحويله لشارة ركن صغيرة (✨ "جديد") في الزاوية العلوية اليمنى أو دمجه بصرياً مع الهيدر بدل الشريط العريض.
- محتوى الكارت (الأكلات + الملاحظات) يبقى كاملاً بدون تقليص.

### 2) تدوير تلقائي للصفحات عند تجاوز السعة (`src/pages/dashboard/KitchenDashboard.tsx`)
- ثابت `ORDERS_PER_PAGE = 18` (6 أعمدة × 3 صفوف، نفس الشبكة الحالية).
- حساب `pages = chunk(activeOrders, 18)` و `totalPages = pages.length`.
- `useState currentPage` مع `setInterval` كل **8 ثوانٍ** يدور للصفحة التالية ثم يرجع للأولى (rotation تلقائي بدون تدخل).
- إيقاف التدوير عندما `totalPages <= 1`.
- إعادة `currentPage` لـ 0 عند تغيّر `totalPages` (لمنع صفحة فارغة).
- إزالة `overflow-auto` من `<main>` (لم تعد هناك حاجة للتمرير).
- إضافة مؤشر صفحات صغير أسفل/أعلى الشبكة (مثل `1/3 • تدوير تلقائي`) داخل `BatchPrepBar` أو فوق الشبكة مباشرة، يظهر فقط إذا `totalPages > 1`.
- transition بسيط (`animate-fade-in`) عند تبديل الصفحة.

### 3) الحفاظ على المنطق
- لا تغيير في الترتيب (الأقدم أولاً).
- لا تغيير في الـ siren / الإعلانات الصوتية / الـ realtime.
- الطلبات المتأخرة (30د+) تستمر بالظهور بشاراتها الحمراء وتدخل ضمن دورة الصفحات بشكل طبيعي.
